* 1. Current contract

| State | Current owner | Evidence | Notes |
|-------+---------------+----------+-------|
| read-only flag | =nelisp-emacs= runtime | =files--buffer-read-only-p=, =nemacs-read-only=, =nemacs-buffer-read-only-store/= | GUI should only render returned state. |
| modified flag | =nelisp-emacs= runtime | =files--buffer-modified-p= and =files--write-redisplay-state= | There is no per-buffer transport store for modified state yet. |
| modeline text | =nelisp-emacs= runtime | =files--write-redisplay-state= writes =nemacs-modeline=. | GUI reads and draws it; it should not synthesize mode-line state. |
| status/refusal | =nelisp-emacs= runtime | =nemacs-status= receives =read-only= / =error= / =permission-denied= / =unsupported=. | GUI displays status text when present. |

Mode-line prefix currently follows this order in runtime:

1. default =--=
2. read-only => =%%=
3. modified => =**=

That means read-only + modified currently displays =**=, not an Emacs-like
=%*=.  It is uncommon in normal editing because read-only edits are rejected,
but it is still observable through explicit modified-state operations or future
commands.

* 4. Small ERT addition plan

Add focused tests to =test/nemacs-gui-file-bridge-runtime-test.el= rather than
expanding the existing large command blocks.

** A. dirty edit / save / write / revert modeline

Purpose: pin the ordinary lifecycle.

Steps:

1. Seed visited file, clean buffer, =nemacs-read-only= =0=.
2. Raw =Z= or direct =self-insert-command=.
3. Assert =nemacs-buf= changed and =nemacs-modeline= starts with =**=.
4. Run =save-buffer=.
5. Assert disk content changed and =nemacs-modeline= starts with =--=.
6. Dirty buffer again, run =write-file= to a second path.
7. Assert second file content, =nemacs-file= second path, =nemacs-modeline= starts with =--=.
8. Dirty buffer again, run =revert-buffer=.
9. Assert buffer equals disk and =nemacs-modeline= starts with =--=.

Expected owner: =nelisp-emacs=.

** B. read-only refusal preserves state and display

Purpose: keep read-only refusal visible and non-mutating.

Steps:

1. =find-file-read-only= a file.
2. Assert =nemacs-read-only= =1= and =nemacs-modeline= starts with =%%=.
3. Attempt =self-insert-command= or raw printable key.
4. Assert buffer unchanged, point policy unchanged or explicitly documented,
   =nemacs-status= =read-only=, =nemacs-read-only= =1=, and modeline still
   starts with =%%=.

Expected owner: =nelisp-emacs= for status/modeline state.  =nelisp-gui= only
needs a visual smoke if status placement is unclear.

** C. dirty flag survives buffer switch

Purpose: expose the missing per-buffer modified state.

Steps:

1. Dirty =main= with =self-insert-command=.
2. Assert =nemacs-modeline= starts with =**=.
3. Switch to =other=.
4. Switch back to =main=.
5. Assert =main= modeline still starts with =**=.

Expected current result: likely fails, because =files--switch-to-buffer= sets
=files--buffer-modified-p= to nil and no =nemacs-buffer-modified-store/= exists.

Expected owner: =nelisp-emacs= semantics.  If fixed through file transport,
=nelisp-gui/bin/nemacs= and =nelisp-gui/nemacs-mx.sh= must initialize
=nemacs-buffer-modified-store/= like the existing read-only store.

** D. read-only + modified mode-line prefix

Purpose: decide and pin the display policy.

Steps:

1. Make current buffer read-only.
2. Force modified state with =not-modified= plus prefix arg or
   =set-buffer-modified-p= inside a runtime probe.
3. Run redisplay.
4. Assert chosen prefix.  For Emacs parity, use =%*=.

Expected owner: =nelisp-emacs=.

* 5. Layer decisions

| Concern | Fix layer | Reason |
|---------+-----------+--------|
| Which commands mark buffers modified | =nelisp-emacs= | This is Emacs command semantics. |
| Save/write/revert clearing modified | =nelisp-emacs= | Runtime owns buffer and visited-file semantics. |
| Per-buffer modified persistence | =nelisp-emacs= primary, =nelisp-gui= transport initialization if a file store is added | The meaning is semantic; launcher/wrapper only create/reset transport files. |
| read-only edit refusal | =nelisp-emacs= | Runtime decides whether a command can mutate text. |
| read-only refusal wording/status | =nelisp-emacs= | Runtime owns =nemacs-status= contents. |
| modeline string contents | =nelisp-emacs= | Runtime writes =nemacs-modeline=. |
| drawing modeline/status text | =nelisp-gui= | GUI should paint returned state without rewriting it. |

* 6. Recommended order

1. Add ERT A and B as passing smoke, because they should match the intended
   current behavior and protect daily-driver UX.
2. Add ERT C as =expected failure= or keep it as a design TODO until
   =nemacs-buffer-modified-store/= is introduced.
3. Decide the read-only dirty prefix policy, then add ERT D.
4. Only after ERT A/B are stable, add one GUI visual smoke that drives real
   runtime transitions instead of directly editing =nemacs-modeline=.
