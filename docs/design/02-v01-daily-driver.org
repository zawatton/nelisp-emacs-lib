* §0 motivation

** 0.1 何故 v0.1 を切るか

Doc 01 §3.4 の Phase 4 (= MELPA helm/ido/dired/org/company/eglot reference
subset、~3000-8000 LOC、6-12ヶ月) を完遂するまで nelisp-emacs +
nelisp-emacs-gtk は daily-driver として使えない。これは「user (=
zawatton) 自身が日常作業で本 binary を Emacs 代替として使い始める」
までの距離が長すぎる。

本 doc は *Phase 4 を deprecate せず、parallel に進めながら* user の
主用途 (= org-mode + elisp 開発 + dired) で daily-driver として使える
最小 MVP を切り出し、~4-5ヶ月で ship する代替 path を定義する。

** 0.2 v0.1 の本質

「Emacs として使える」を *user 主観で* 切り取る:
- =journals-2026.org= / =todo.org= を nemacs-gtk 上で編集できる
- =anvil-*= elisp を =C-x C-e= で eval、=C-h f= で describe できる
- =M-x dired= で directory を辿って file 開ける
- =M-!= で shell command 実行できる

Phase 4 が目指す「helm で fuzzy 補完」「magit で git 操作」「eglot で
LSP」等の発展機能は v0.1 では out-of-scope (= Phase 4 / external repo
担当)。

* §3 milestone 詳細

** 3.1 M1 — file/buffer + dired-min (4-5w)

*scope*:

1. find-file
   - =find-file (FILENAME)= interactive (=C-x C-f=)
   - minibuffer prompt =Find file: = + path completion (= built-in
     =read-file-name=、helm/ido 不要)
   - 既存 buffer に同 file あれば switch-to-buffer、なければ buffer 作成
     + =insert-file-contents= で content load
   - =buffer-file-name= / =default-directory= 設定
   - major-mode dispatch (= =auto-mode-alist= 引き、= =fundamental-mode=
     fallback)
2. find-file-noselect (= programmatic、interactive ではない)
3. save-buffer
   - =save-buffer= interactive (=C-x C-s=)
   - =buffer-file-name= に =write-region= で書き戻し
   - =buffer-modified-p= が nil なら no-op (+ message)
   - coding system 既定は =utf-8= (= NeLisp Doc 38 coding contract に従う)
4. switch-to-buffer / kill-buffer / list-buffers
   - =switch-to-buffer (BUFFER-OR-NAME)= interactive (=C-x b=)
     - minibuffer + =completing-read= で buffer 名選択
     - 存在しない名前なら新 buffer を作成
   - =kill-buffer (BUFFER)= interactive (=C-x k=)
     - buffer-modified-p なら =yes-or-no-p= で確認
   - =list-buffers= interactive (=C-x C-b=)
     - =*Buffer List*= buffer 描画 (= name / size / mode / file 4 列)
5. window split / navigate
   - =split-window-below= (=C-x 2=) / =split-window-right= (=C-x 3=)
   - =other-window N= (=C-x o=) — 順巡回
   - =delete-window= (=C-x 0=) / =delete-other-windows= (=C-x 1=)
   - 既存 =emacs-window.el= の stub を real impl に置換
6. dired-min
   - =M-x dired= → minibuffer prompt =Dired (directory): = → =*Dired DIR*= buffer
   - =directory-files-and-attributes= で listing、簡易 1-line/file 描画
   - keybinding (= =dired-mode-map=):
     | key | command | 内容 |
     |-----+---------+------|
     | =RET= | dired-find-file | point の file/dir を find-file |
     | =n= | dired-next-line | next line |
     | =p= | dired-previous-line | previous line |
     | =q= | quit-window | dired buffer を bury |
     | =g= | revert-buffer | listing 再読込 |
     | =^= | dired-up-directory | 親 dir に move |
   - mark / copy / rename / chmod / sort 等は **scope 外** (= Phase 4)

*close gate*:

** 3.2 M2 — 開発体験 (3-4w)

*scope*:

- isearch-forward / isearch-backward (= =C-s= / =C-r=) interactive UI、minibuffer prompt + match highlighting (= overlay or face、既 =emacs-faces.el= に依存)
- undo (= =C-/=) / undo-redo (= =C-x u=) interactive layer (= 既 =emacs-undo.el= primitive 上)
- *help system*:
  - =describe-function= (=C-h f=) → =*Help*= buffer に signature + docstring
  - =describe-variable= (=C-h v=) → value + docstring
  - =describe-key= (=C-h k=) → keybinding 解析
  - =help-mode= (= =q= で quit、=l= で history back、=TAB= で next-button)
- eval-defun (=C-x C-e= = eval last sexp、=C-M-x= = eval defun)
- eval-region / eval-buffer
- =M-x ielm= = comint-style elisp REPL minimum

*close gate*:

| # | gate |
|---+------|
| 1 | =C-s foo= で buffer 内 first match に point move、=C-s= 連打で next、=C-g= で abort & 元に戻る |
| 2 | =C-/= で 1 step undo、=C-x u= で redo |
| 3 | =C-h f find-file= → =*Help*= に signature + docstring 表示 |
| 4 | =C-h k C-x C-f= → "C-x C-f runs find-file" 様の表示 |
| 5 | =(defun foo () 42)= を =C-x C-e= で eval、ミニバッファ echo 42 |
| 6 | =M-x ielm= で REPL prompt、=(+ 1 2)= → 3 |

*LOC 見積*: 800-1200

** 3.3 M3 — org-mode subset (6-8w)

*scope* (= user の =journals-YYYY.org= / =todo.org= / =capture/= 用途 subset):

- outline structure
  - =org-mode= major-mode (= file extension =.org= で auto-dispatch)
  - heading promote/demote (=M-←= / =M-→=)
  - insert-heading (=M-RET=)
  - subtree fold/unfold (= =TAB= / =S-TAB= / =org-cycle-global=)
- TODO state cycle
  - =org-todo= (=C-c C-t=) で keyword cycle
  - todo.org の 6 keyword 対応 (= INBOX / NEXT / WAIT / PROJECTS / SCHEDULED / SOMEDAY)
  - DONE marker + log 軽量
- table edit
  - =org-table-align= (=TAB= within table)
  - =org-table-next-field= (=TAB=)
  - column add/remove は最低限
- agenda 軽量
  - =org-agenda= (=C-c a a=) で SCHEDULED + DEADLINE 一覧
  - 日付 group / TODO state filter
  - dispatcher は =a= (agenda) / =t= (todo) / =m= (match) のみ、=L= / =B= 等の advanced は scope 外
- capture 軽量
  - =org-capture= (=C-c c=)
  - template DSL: =(file+headline FILE HEADLINE)= / =(file+olp+datetree FILE …)= の 2 種のみ実装
  - user 既存 template (Journal / Todo) が動くこと

*scope 外* (= Phase 4 担当):

- org-publish / org-roam / org-noter
- agenda full (= block / weekly / category dispatcher 全種)
- src block babel execute / tangle
- :PROPERTIES: drawer 高度処理 (logbook 全機能、clocking 等は最低限)

*close gate*:

| # | gate |
|---+------|
| 1 | =todo.org= を nemacs-gtk で開くと outline 構造が表示、=TAB= で fold/unfold |
| 2 | TODO heading で =C-c C-t= → state cycle、INBOX → NEXT → WAIT → … |
| 3 | =journals-2026.org= の datetree に =C-c c j= で entry 追加できる |
| 4 | =C-c a a= で SCHEDULED 一覧 agenda buffer 表示 |
| 5 | table 上で =TAB= でセル間移動 + auto-align |

*LOC 見積*: 2500-3500 (= 本 v0.1 の最大 milestone)

** 3.4 M4 — shell + completion (3-4w)

*scope*:

- shell-command (=M-!=) / shell-command-on-region (=M-|=)
  - output → =*Shell Output*= buffer、最低限 process 同期実行
  - 既 =emacs-process.el= 上に layer
- async-shell-command (=M-&=) — 非同期、=*Async Shell Command*= buffer
- minibuffer completion (= built-in =completing-read= 拡張)
  - =TAB= で completion candidates 出す
  - =*Completions*= buffer 描画
  - icomplete style live-update は **optional / 後送り**
- project.el subset
  - =project-current= = =vc-root-of-buffer= or =.git= directory 検出
  - =project-find-file= = =find-file= の root 限定版
  - =project-switch-project= = 既知 project list から選択

*close gate*:

* §7 open question / 後送り

** 7.1 M2 help-mode のナビゲーション粒度

- =l= (history back) / =b= (back symbol) / =TAB= (next button) を v0.1 で
  どこまで実装するか

** 7.2 M3 capture template DSL の互換性

- v0.1 で =file+headline= / =file+olp+datetree= の 2 形式のみ

  flag は v0.1 で対応するか後送りか

** 7.3 M4 minibuffer completion style

- icomplete style (= live-update inline 候補) を v0.1 で実装するか

** 7.4 M3 org-mode の =emacs-org-subset.el= vs vendored Org

- 既存 Org Mode の lisp ファイル (= =org.el= ~30k LOC) を vendor して
  ロードする path も検討余地あり
- 案: vendored は外部依存が大きい (= cl-lib / org-macs / ob-* 多数)、subset 自前実装の方が
  v0.1 では現実的。Phase 4 で full vendor 検討

  Emacs default の =-al= ls と一致しない可能性
- 案: v0.1 は alphabetical 固定、=-al= 互換は Phase 4
