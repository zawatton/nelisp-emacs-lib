* 0. Purpose

This is the package ownership inventory for the =nelisp-emacs-libs=
library distribution phase.  It assigns every current =src/*.el= file to
a logical package group without moving files.

The current repository may still contain application and GUI files from
the historical =nelisp-emacs= / =nemacs= work.  Those files are extraction
inputs for =nemacs-next= unless they are explicitly classified as reusable
library packages.

The goal is coordination, not churn.  Multiple sessions may be editing
the tree at the same time, so this inventory should guide new work while
avoiding broad renames, mass moves, or formatting-only changes.

* 1. Ownership groups

| Group | Future package | Owns | Must not depend on |
|-------+----------------+------+--------------------|
| FND | =packages/foundation= | pure Elisp primitives, data helpers, host-safe compatibility basics | buffer/window/frame, GUI/TUI, app entrypoints |
| TXT | =packages/text-intl= | coding systems, char tables, international/text data helpers | app entrypoints, GUI/TUI |
| BUF | =packages/buffer= | buffer model, point, text properties, search/edit/undo primitives | concrete GUI/TUI transports |
| CORE | =packages/editor-core= | command-loop, keymap, minibuffer, window/frame semantics, modes/faces/help/info | concrete GUI transport, =nemacs-main= |
| IO | =packages/io= | file/process/network/sqlite/runtime adapters | app shortcuts, GUI-owned behavior |
| DSP | =packages/display= | redisplay model, TUI backend, terminal event/terminfo | GUI app code, command semantics |
| FEAT | =packages/features= | reusable user-facing Elisp feature subsets such as dired/org/project/shell/vc | private app glue |
| PKG | =packages/package-compat= | package/vendor/MELPA compatibility and facade shims | app entrypoints |
| APP | =nemacs-next/apps/nemacs= | runnable editor assembly and bootstrap extraction input | reusable semantics not already in packages |
| GUI | =nemacs-next/frontends= | GUI/TUI consumer, transport, rendering, launch/install extraction input | shared command semantics |
| LEGACY | =legacy-or-spike= | temporary compatibility, residual stubs, spike code | permanent public API |

* 2. Rules for using this inventory

- New reusable behavior goes into the owner group, even if the current
  file path remains under =src/=.
- Do not move files solely to satisfy this inventory while other sessions
  are active.
- If a GUI/TUI/app file needs new command behavior, add that behavior to
  BUF/CORE/FEAT first and call it from the adapter.
- APP/GUI owners are transitional.  New reusable behavior must not be added
  there merely because the current caller is =bin/nemacs= or GUI bridge
  code.  Treat those changes as =nemacs-next= extraction work.
- Public reusable APIs should be prefixed.  Unprefixed Emacs-compatible
  names belong in explicit builtins/shim files.
- Cross-module calls to =--= helpers are not stable APIs.  Promote only
  the helpers needed by real consumers.
- Update this inventory whenever a file's ownership changes.

* 5. Immediate work queue

** 5.1 Low-conflict tasks

These are safe while multiple sessions are active:

1. Add package group comments to docs only.
2. Add focused tests around existing public APIs.
3. Add anvil-based reports for cross-module private helper use.
4. Update GUI docs to point at Doc 17/18.
5. Add Makefile targets that run existing tests by group without moving
   files.

** 5.2 Tasks to avoid during parallel sessions

Avoid these until active branches are coordinated:

1. bulk moving =src/*.el= into =packages/=,
2. renaming public features,
3. changing =provide= names,
4. reformatting large files,
5. replacing app launch paths used by current gates,
6. changing GUI transport file locations without wrapper compatibility.

* 6. First enforcement targets

The first enforcement should be advisory, not blocking:

- report files that call private helpers across ownership groups,
- report APP/GUI files that define command semantics,
- report package groups pulled in by =emacs-init.el=,
- report =nemacs-main.el= direct shortcuts that should move to FEAT/CORE,
- report GUI fallbacks that duplicate shared runtime behavior.

Once reports are stable, selected checks can move into
=make gate-nemacs-complete=.

** 6.2 Ownership coverage target

=make nemacs-ownership-coverage= writes:

- =build/nemacs-ownership-coverage.tsv=
- =build/nemacs-ownership-coverage-summary.org=

The coverage target verifies that every =src/= and =gui/= Elisp file is
covered by this document.  It also reports exact Doc 18 file rows that no
longer exist.  =make nemacs-library-gate= requires both =unowned= and
=stale= rows to remain at zero.

** 6.3 Advisory report target

=make nemacs-library-boundary-report= writes:

- =build/nemacs-library-boundary-report.tsv=
- =build/nemacs-library-boundary-summary.org=

The report currently covers:

- =private-cross-owner=: a call to a defined =--= helper whose provider
  group differs from the caller group;
- =app-gui-semantic-definition=: an APP/GUI definition whose name looks
  like reusable command/editor semantics;
- =emacs-init-require=: current package groups pulled by
  =src/emacs-init.el=.

This target is intentionally advisory.  It should be used to choose small
API-promotion and adapter-cleanup tasks, not to block parallel sessions.
