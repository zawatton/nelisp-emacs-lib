#+title: Small Vendor Foundation Candidates
#+date: 2026-06-03

This note lists small vendored Emacs Lisp files that are good candidates
to enable before attempting large dependency surfaces such as
=org-element.el=.  The goal is to widen the standalone REPL/vendor load
surface from dependency foundations upward.

* Priority candidates

| Candidate | Lines | Main dependencies | Status | Reason |
|-----------+-------+-------------------+--------+--------|
| =vendor/emacs-lisp/format-spec.el= | 205 | none | default replay | Direct dependency of =org-macs.el=.  Standalone form-walk, true-load, and persistent REPL diagnostics pass. |
| =vendor/emacs-lisp/org/org-version.el= | 24 | none | default replay | Tiny base file used around Org compatibility/version surfaces.  True-load and persistent REPL prove =org-release= / =org-git-version= function surface. |
| =vendor/emacs-lisp/org/org-macs.el= | 1821 | =cl-lib=, =format-spec=, =org-version= | default replay | Central Org macro/helper layer depended on by many remaining Org files. |
| =vendor/emacs-lisp/org/org-compat.el= | 1822 | =cl-lib=, =seq=, =org-macs= | default replay | Compatibility layer for many Org follow-up modules. |
