* 0. Purpose

This document defines the release-only workflow for creating detached
signatures for generated package archive artifacts.

Normal library development keeps package archives unsigned.  Public release
publication requires detached signatures for:

- =build/nemacs-library-package-archives/archive-contents=
- every =build/nemacs-library-package-archives/*.tar= package archive

The signature artifact set is generated by:

#+begin_src sh
make nemacs-library-package-signature-policy
#+end_src

Current draft policy coverage is 9 artifacts: one =archive-contents= file
and 8 package tarballs.

* 1. Signing

Release signing is explicit and not part of =make nemacs-library-gate=:

#+begin_src sh
make nemacs-library-package-signature-release-sign \
  NEMACS_LIBRARY_RELEASE_SIGNING_KEY_FINGERPRINT=0123456789ABCDEF0123456789ABCDEF01234567
#+end_src

The signing target writes:

- =build/nemacs-library-package-signature-release-sign.tsv=
- =build/nemacs-library-package-signature-release-sign.org=

It creates the detached signature files recorded by the signature policy TSV.
The default suffix is =.sig=.  The target uses:

#+begin_src sh
gpg --batch --yes --local-user FINGERPRINT --output ARTIFACT.sig --detach-sign ARTIFACT
#+end_src

Set =NEMACS_LIBRARY_RELEASE_SIGNATURE_ARMOR= to =1= if a release process
requires armored detached signatures while keeping the configured suffix.
Set =NEMACS_LIBRARY_RELEASE_GPG_PROGRAM= to use a non-default GnuPG binary.

* 2. Verification

After signing, verify with:

#+begin_src sh
make nemacs-library-package-signature-release-verify \
  NEMACS_LIBRARY_RELEASE_SIGNING_KEY_FINGERPRINT=0123456789ABCDEF0123456789ABCDEF01234567
#+end_src

The strict verification target writes:

- =build/nemacs-library-package-signature-release.tsv=
- =build/nemacs-library-package-signature-release.org=

It requires:

- =NEMACS_LIBRARY_RELEASE_SIGNING_KEY_FINGERPRINT= to be a 40-hex OpenPGP
  fingerprint;
- every expected detached signature file to exist;
- =gpg --verify SIGNATURE ARTIFACT= to succeed;
- the =VALIDSIG= fingerprint emitted by GnuPG to match the configured
  fingerprint.

The combined release workflow is:

#+begin_src sh
make nemacs-library-package-signature-release \
  NEMACS_LIBRARY_RELEASE_SIGNING_KEY_FINGERPRINT=0123456789ABCDEF0123456789ABCDEF01234567
#+end_src

This runs release signing and strict verification in order.

* 3. Release Bundle

Normal development can retain a draft bundle and manifest without private key
access:

#+begin_src sh
make nemacs-library-package-release-bundle-manifest
#+end_src

This writes:

- =build/nemacs-library-package-release-bundle/=
- =build/nemacs-library-package-release-bundle-manifest.tsv=
- =build/nemacs-library-package-release-bundle-manifest.org=

Draft mode copies =archive-contents=, package tarballs, and publication
evidence, then records missing signatures and release-only evidence as pending.

For a public release, create the strict signed bundle with:

#+begin_src sh
make nemacs-library-package-release-bundle \
  NEMACS_LIBRARY_RELEASE_SIGNING_KEY_FINGERPRINT=0123456789ABCDEF0123456789ABCDEF01234567
#+end_src

This target runs release signing, strict signature verification, and strict
bundle retention in order.  Missing signatures or release verification
evidence fail the target.

* 4. Current State

The repository currently verifies the workflow shape but does not contain
release private key material or committed release signatures.  Running
=make nemacs-library-package-signature-release-sign= without a configured
fingerprint fails with 9 signing failures.  Running strict verification
without the signature files fails with 9 verification failures.

This is intentional: release signing requires explicit key access and must not
happen as part of normal local development or the reusable-library gate.

* 5. Release Checklist

For a real public release:

- run =make nemacs-library-gate= first;
- run =make nemacs-library-package-release-bundle= with the release key
  fingerprint;
- retain =build/nemacs-library-package-release-bundle/=,
  =build/nemacs-library-package-release-bundle-manifest.tsv=, and
  =build/nemacs-library-package-release-bundle-manifest.org= together;
- publish only if strict signature verification reports 9 verified signatures
  and 0 failures, and strict bundle retention reports 0 failures.
