* 守ること

- publish settle を 1ms に戻して短縮したことにしない。
- host 側で session/progn を単純に一括化しない。
- writeback dispatch table / wrapper 型 minimal read path には戻らない。
- command ごとの writeback state を理解せずに fast path を追加しない。
- 対象外の user 変更を revert / commit しない。
- 1 commit 1 chunk を維持する。
