* 0. Purpose

This document records whether the current library package descriptors and
layout plan are ready for a physical =packages/= extraction pass.

The current answer is: the generated package shape is coherent enough to
start a staged extraction experiment, but the source tree should not be
moved in one large change.  The first physical move should keep =src/= as
the authoritative compatibility load path or provide compatibility
forwarders, because much of the repository, tests, scripts, and external
habits still expect =src/= paths.

  changes and remove generated =*.elc= files from =src/=, =scripts/=, and
  =gui/= after gate runs;

  =src/= compatibility forwarders or an equivalent migration bridge;
- run the real release path through =docs/release/nemacs-library-release.local.mk=
  or =NEMACS_LIBRARY_RELEASE_CONFIG= in a controlled environment, replacing
  the throwaway rehearsal key with real release key material;
- keep =nelisp-gui= and app/frontends as consumers of reusable packages, not
  owners of core command, buffer, window, keymap, minibuffer, undo, file, or
  text semantics.

For AI agents: prefer continuing from the evidence targets listed below
instead of inventing new readiness criteria.  When a count changes, update

* 3. Readiness Decision

** 3.1 Ready

These properties are ready enough for a staged physical extraction:

- package membership is generated from the facade manifest, not manually
  guessed;
- package dependencies resolve to known facade packages, host features, or
  vendored packages;
- unknown external features are empty;
- hidden reusable dependencies and migration candidates are empty;
- lazy package companions are explicit;
- stable package API and stable lazy API symbols resolve;
- stable API compatibility policy classifies all stable symbols as preferred
  =public-prefixed= library API or intentional =compat-global= shim API;
- residual non-stable APIs are reviewed in Doc 26;
- boundary report rows are zero;
- reusable package scaffold files do not require app/frontend scaffold
  features;
- package archive style metadata drafts exist for every reusable package;
- every reusable package has an isolated package-style install/load smoke
  from generated install directories without =src/= fallback;
- every reusable package has a generated package tar archive and
  install-from-archive smoke through Emacs =package-install-file=;
- generated package tar archives rebuild reproducibly with stable SHA-256
  checksums;
- every reusable package is discoverable through generated
  =archive-contents= and installable through Emacs =package-refresh-contents=
  plus =package-install=;
- publication policy checks cover shared package versioning, checksum
  retention, archive index retention, local unsigned development archives,
  and external detached-signature requirements;
- release public key policy records the expected public key file and
  fingerprint checks in draft mode and fails closed in strict mode when key
  material or the expected fingerprint is missing;
- signature policy checks record every release artifact that needs a detached
  signature and provide a strict release mode that imports the explicit
  release public key into a temporary GnuPG keyring before verifying signer
  fingerprint matches;
- release signing workflow can create detached signatures for every release
  artifact with an explicit configured OpenPGP key fingerprint;
- release bundle retention manifest copies draft archive/evidence files and
  records release-only signatures/evidence as pending without hiding the
  public release gap;
- release bundle consumer smoke verifies retained manifest byte/hash values
  and installs every reusable package from the retained bundle archive
  directory;
- release rehearsal generates a throwaway OpenPGP key under =build/=, signs
  rehearsal-local archives, verifies 9 detached signatures against the
  exported public key, retains a strict bundle with no pending files, and
  installs every reusable package from that signed bundle;
- dependency publication policy checks cover package-owned lazy companions,
  host-provided features, vendored nelisp package dependencies, and empty
  unpublished dependency relation buckets;
- package lazy metadata records direct and additional package dependency
  closure for every lazy companion without app/frontend or unknown edges;
- vendored dependency release verification records the vendor Git HEAD and
  package directory SHA-256 for =nelisp-process= and verifies the package path
  is clean in strict release mode;
- package smoke tests require every loader feature independently.

** 3.2 Not Ready For One-Shot Move

The repository is not ready for a one-shot move that removes =src/= as the
primary load path.  Reasons:

- tests and scripts still assume =emacs -Q -L src=;
- many generated artifacts describe future paths but do not rewrite
  =require= or =load-path= behavior;
- application/front-end code outside the facade package set still consumes
  current =src/= feature locations;
- host users and parallel sessions may have local workflows tied to =src/=;
- package archive metadata, per-package README files, deterministic
checksum verification, archive index, publication policy, release public key
policy, release signature target verification, release signing workflow
targets, draft release bundle retention manifest, and install tests exist as
generated drafts, and a throwaway-key strict release rehearsal proves the
mechanics of signing, verification, bundle retention, and bundle smoke.  Actual
release public key material, release signing keys, retained real detached
signature artifacts, and public publication workflow execution do not exist yet.

The correct next step is a compatibility-preserving extraction scaffold,
not a destructive file move.

The scaffold should include:

- copied source files for local experiment only;
- an umbrella =packages/nelisp-emacs-facade/lisp/nelisp-emacs.el=

- a per-package README generated from the package guide and Doc 26
  residual decisions;
- a per-package smoke command that uses only that package's lisp path plus
  declared package dependencies.

App/bootstrap/frontend glue that is not yet reusable package material is staged by
=make nemacs-library-app-scaffold= under
=packages/nelisp-emacs-app-gui/=.  This scaffold is explicitly outside the
descriptor package count and exists to make package-path GUI bridge runtime
images prefer mapped app/frontend copies instead of implicit =src/=
fallbacks.  App-local utilities such as =image-baker= are staged here, not
in a reusable package, because they invoke bootstrap policy through
=nemacs-loadup=.

=make nemacs-library-app-boundary= verifies the app/bootstrap contract:
=emacs-init=, =nemacs-loadup=, =nemacs-main=, and =image-baker= must be
present in the app scaffold and absent from reusable package scaffolds.
It also verifies that reusable lisp-image helpers =emacs-dump= and
=image-loader= are no longer staged as app-only files.

=make nemacs-library-package-app-require-guard= verifies the inverse
require contract on the generated scaffold: reusable package files may not
=require= features provided by =packages/nelisp-emacs-app-gui/=.  Current
coverage scans 88 package files, indexes 10 app/frontend features, scans
130 package require edges, and reports 0 violations.

=make nemacs-library-package-metadata= writes package archive style draft
metadata for all eight reusable packages, including summary, keywords,
project URL, =GPL-3.0-or-later= license, maintainer/author, package
dependencies, loader feature, and generated =packages/PACKAGE/PACKAGE-pkg.el=
files.  =make nemacs-library-package-install-smoke= then runs each reusable
package in a separate =emacs -Q= process through a temporary package-style
install root, flattening the eager/lazy scaffold files into
=PACKAGE-VERSION/= directories and requiring the package loader with only
its dependency closure available.  Current install smoke coverage is 8
packages, 8 ok, 0 failures, and no =src/= or generated =packages/= source
leaks.

=make nemacs-library-package-archive= builds draft multi-file package tar
archives under =build/nemacs-library-package-archives/= from the generated
metadata and package scaffold.  =make nemacs-library-package-archive-smoke=
then installs each package and its dependency closure through Emacs
=package-install-file= in a fresh package user directory and requires the
loader feature from the installed archive output.  Current archive smoke
coverage is 8 packages, 8 ok, 0 failures, and no =src/= or generated
=packages/= source leaks.  Package install still emits byte-compile warnings
from the current source surface; these are not treated as archive load
failures unless the loader/member feature proof fails.

=make nemacs-library-package-archive-checksum= rebuilds every package archive
twice in isolated roots and compares those rebuilds with the primary archive
SHA-256 digest.  Current checksum coverage is 8 packages, 8 deterministic, and
0 failures.  Archive generation fixes tar entry order, owner/group metadata,
numeric ownership, and mtime so local rebuilds produce stable tar digests.

=make nemacs-library-package-archive-index= writes a local package archive
=archive-contents= file next to the generated tarballs.  =make
nemacs-library-package-index-smoke= configures a fresh =package-user-dir=,
runs =package-refresh-contents= against that local archive location, installs
each reusable package through =package-install=, verifies Emacs resolved the
declared package dependencies, requires the loader/member features, and checks
that no repo =src/= or generated =packages/= scaffold file was loaded.  Current
index smoke coverage is 8 packages, 8 ok, and 0 failures.  The local archive
location is passed to =package-archives= as an absolute directory name, because
Emacs =package.el= accepts either =http(s)= URLs or absolute local file names
for this path.

=make nemacs-library-package-publication-policy= records and verifies the
current publication policy.  Current coverage is 35 checks, 35 ok, and 0
failures.  The policy keeps local development archives unsigned, requires
public release archives to retain SHA-256 checksums for every tarball, requires
detached signatures for public =archive-contents= and tarballs, verifies every
retained archive is still referenced by the archive index, and enforces one
shared =X.Y.Z= package version while facade packages are versioned together.
The target records signature requirements only; signature artifact targets and
strict release verification are covered by the dedicated signature policy gate.

=make nemacs-library-package-release-key-policy= records the public key policy
for release verification.  Current draft coverage is 4 checks: expected
fingerprint, public key file, public key import, and fingerprint match.  Without
=NEMACS_LIBRARY_RELEASE_SIGNING_KEY_FINGERPRINT= and
=docs/release/nemacs-library-release-public-key.asc=, draft mode records 4
pending rows and 0 failures.  Strict mode is release-only and fails closed
unless the public key file is readable, importable by GnuPG, and contains the
configured 40-hex OpenPGP fingerprint.

=make nemacs-library-package-signature-policy= records the release signature
artifact set for public package publication.  Current draft coverage is 9
artifacts requiring detached signatures: one =archive-contents= file and
8 package tarballs.  Draft mode records expected =.sig= paths, reports
0 signature files present, and has 0 failures.  The release-only strict target
=make nemacs-library-package-signature-release-verify= requires
=NEMACS_LIBRARY_RELEASE_SIGNING_KEY_FINGERPRINT= to be a 40-hex OpenPGP
fingerprint, requires the explicit release public key file, imports that key
into a temporary GnuPG keyring, requires detached signatures for every
artifact, runs =gpg --verify=, and verifies the =VALIDSIG= fingerprint matches
the configured key.  This repository does not yet create or retain release
signatures; the strict target is the release gate for those future artifacts.

=make nemacs-library-package-signature-release-sign= creates the detached
signature artifacts listed by the signature policy TSV.  It is release-only
and intentionally outside =make nemacs-library-gate= because it requires local
private key access.  It requires
=NEMACS_LIBRARY_RELEASE_SIGNING_KEY_FINGERPRINT=, signs =archive-contents= and
all 8 package tarballs with =gpg --detach-sign=, and records signature byte
sizes and SHA-256 digests.  Without a configured release key it fails closed;
the current no-key check reports 9 signing failures and creates no publishable
release signature set.

=make nemacs-library-package-release-bundle-manifest= retains the draft
publication bundle under
=build/nemacs-library-package-release-bundle/= and writes a manifest for every
package archive, expected detached signature, and publication evidence file.
Draft mode is part of =make nemacs-library-gate=: it copies unsigned archives
and policy evidence, including release public key policy evidence, while
recording missing release-only public key material, signatures, and release
verification evidence as pending.  Current draft coverage is 35 rows, 21
retained files, 14 pending release-only files, and 0 failures.  =make
nemacs-library-package-release-bundle= is release-only; it runs release signing
and strict signature verification first, then regenerates the bundle manifest in
strict mode so missing public key material, signatures, or release evidence
fail.

=make nemacs-library-package-release-rehearsal= generates a throwaway GnuPG
key under =build/nemacs-library-package-release-rehearsal/gnupg=, exports the
public key, rebuilds package archives under the rehearsal root, signs the
rehearsal-local =archive-contents= and 8 tarballs, verifies every detached
signature through the strict public-key verifier, retains a strict signed
bundle, and runs the retained-bundle package install smoke.  Current rehearsal
coverage is release key policy ok 4, signing failures 0, signature
verification failures 0, bundle retained files 35, bundle pending files 0,
bundle failures 0, bundle smoke ok 8, and bundle smoke failures 0.  The
rehearsal key is intentionally not a public trust anchor.

=make nemacs-library-package-release-ready= verifies the real-release local
config path before a public signing run.  The local config is intentionally
ignored by git and must provide the release public key file, the full 40-hex
OpenPGP fingerprint, and an external =NEMACS_LIBRARY_RELEASE_GNUPGHOME= that
contains the configured signing secret key.  The public release entrypoint is
=make nemacs-library-package-release-from-config=, which runs the readiness
check before strict signing, strict verification, strict bundle retention, and
bundle smoke.

=make nemacs-library-package-release-publication-policy= verifies the retained
release bundle before publication.  Draft mode permits only the expected
release-only pending rows, rechecks every retained manifest file by byte count
and SHA-256, requires 8 bundle install smokes to pass, and fails if the bundle
root contains any manifest-external file.  It also fails if retained manifest
rows map multiple evidence subjects to the same bundle path.  Current draft
coverage is 32 checks, 29 ok, 3 pending, and 0 failures.  Strict release mode
requires zero pending rows, all 9 detached signatures, all 5 release-evidence
files, no duplicate bundle paths, no manifest-external files, and successful
bundle smoke.

=make nemacs-library-compat-api-policy= records the consumer-facing stable API

948 stable API rows, 935 =public-prefixed= symbols, 13 =compat-global=

Emacs-compatible shim names.  Private helpers, app/frontend adapters, and
inventory-missing symbols fail the gate if they appear in a stable manifest.

=package-requires=.  Current coverage is 29 checks, 29 ok, and 0 failures:

=lazy/= directory, host features =keymap= and =pp= are explicitly classified
as host Emacs provided features, vendored =nelisp-process= is declared as a

=make nemacs-library-package-lazy-metadata= records reusable dependency

** 4.3 Stage C: Source Of Truth Decision

After Stage B, choose one of two source-of-truth models:

| Model | Meaning | Tradeoff |
|-------+---------+----------|
| =src-primary= | keep editing in =src/= and generate/copy package trees | safest during library API churn |
| =packages-primary= | move real sources under =packages/= and keep =src/= compatibility forwarders | closer to publication, higher churn |

Do not choose =packages-primary= until the package-path gate is as strong
as =make nemacs-library-gate=.
