* 3. Fidelity Table

| API | Fidelity now | Gap |
|-----+--------------+-----|
| =call-process= | Real spawn on standalone through reader process primitives; captures output into current buffer, named buffer, buffer object, or =(:file FILE)=; returns exit code. | Separate stderr routing in =(REAL-BUFFER STDERR-FILE)= is not implemented because the current reader capture pipe is merged. DISPLAY is ignored. Large streaming output still depends on the reader buffer capacity. |
| =process-file= | Local case delegates to =call-process= after handler dispatch. | Remote handlers are not implemented here; Tramp will supply them. |
| =shell-command-to-string= | Already routes through =call-process= when host helper is absent. | Same stderr/coding limits as =call-process=. |
| =make-process= / =start-process= | Existing source path can start output-oriented native reader processes and fire filters/sentinels during =accept-process-output=. Runtime preload has a stronger bidi smoke. | Package-source async still lacks a reusable stdin write path for native reader objects. |
