#+TITLE: nelisp-gui — NeLisp 純elisp/native GUI・ディスプレイ層 (nemacs editor)
#+OPTIONS: toc:2

NeLisp standalone 上で動く純elisp/native-compiled の **GUI / ディスプレイ層**。
X11 wire protocol を *生で喋り*（GTK / Xlib / libX11 非依存、UNIX socket + バイト列）、
Phase-47/aot compiler（=dev/nelisp/lisp/nelisp-aot-compiler.el=）で elisp(低層IR)→native
static ELF にした **nemacs エディタ**を提供する。

=dev/nelisp-ffi=（FFI = .so dlopen/dlsym、SDL バインド）とは別の関心事（描画・UI）。

* 使い方
#+begin_src sh
  dev/nelisp-gui/bin/nemacs                              # 起動
  dev/nelisp-gui/bin/nemacs path/to/file.el              # ファイルを開く
  PATH=dev/nelisp-gui/bin:$PATH emacs path/to/file.el    # emacs 互換入口
  PATH=dev/nelisp-gui/bin:$PATH emacs +42 path/to/file.el # 指定位置を初期 point に反映
  dev/nelisp-gui/scripts/install-user-bin.sh install     # ~/.local/bin に symlink
  dev/nelisp-gui/scripts/install-user-bin.sh uninstall
  dev/nelisp-gui/scripts/install-user-bin.sh rollback
  dev/nelisp-gui/nemacs-build.sh nemacs-editor.el xfont-sexp   # 再ビルド
  NEMACS_SYNC_NELISP=1 dev/nelisp-gui/nemacs-build.sh nemacs-editor.el xfont-sexp
  dev/nelisp-gui/scripts/doctor-workspace.sh             # sibling repo 状態確認
  dev/nelisp-gui/scripts/verify-nemacs-gui.sh            # 連携 smoke
  dev/nelisp-gui/scripts/stress-nemacs-session.sh 120    # session bridge 耐久 smoke
  make -C dev/nelisp-emacs test-nemacs-gui-bridge NELISP_BIN=/tmp/nelisp-snap/nelisp
#+end_src

必須依存:
| 依存 | 用途 |
|------+------|
| =nelisp= runtime | 初期設定の =nemacs-*= 評価、bridge runtime、通常起動 |
| bootstrap Emacs | =nelisp= snapshot / native GUI binary の build host（runtime fallback ではない） |
| =make= | =nelisp= standalone reader build |
| =python3= | =--goto= / =+LINE[:COL]= の byte offset 解決、設定 binary 生成補助 |
| =awk= / =sed= / =grep= / =tr= | wrapper 内の設定・transport text 処理 |
| =cp= / =rm= / =mkdir= / =chmod= / =readlink= | snapshot 同期、transport 初期化、install 状態確認 |
| X11 =DISPLAY= | native GUI 表示先（doctor では warn） |

設定: =~/.nemacs.d/early-init.el=（frame geometry / display-mode）, =init.el=（theme / font /
=nemacs-initial-buffer=）。launcher は起動時に初期バッファを =/tmp/nemacs-init-buf= へ、
runtime bridge を =/tmp/nemacs-mx.sh= へ配置してから native binary を起動する。
起動時には raw key / direct fallback / minibuffer display / modeline / cursor の
=/tmp/nemacs-*= transport file と buffer list / current buffer name / visited filename
を初期化し、前回 session の stale state を描かない。
ファイル指定時は内容を初期バッファに読み込む。起動後は =C-x C-f= でファイルを開き、
=C-x C-s= / M-x =save-buffer= が元ファイルへ書き戻す。=C-x C-w= は
現在バッファを別名保存し、以後の save 先もそのファイルに切り替える。=C-x C-v= は
現在バッファを別ファイルへ差し替える。=C-x C-r= は read-only buffer として開き、
=C-x C-q= は read-only 状態を切り替える。=C-x i= は minibuffer で指定した
ファイル内容を現在バッファへ挿入する。=C-x s= は既知の visited buffer を
まとめて保存する。

=NEMACS_*= / =NELISP_*= 環境変数:
| 変数 | default | 用途 |
|------+---------+------|
| =NELISP_ROOT= | sibling =../nelisp= など | =nelisp= repo を明示 |
| =NELISP_SNAP= | =/tmp/nelisp-snap= | launcher / build / bridge が使う runtime snapshot |
| =NEMACS_EMACS_ROOT= | sibling =../nelisp-emacs= | =nelisp-emacs= repo を明示 |
| =NEMACS_TRANSPORT_DIR= | =/tmp= | =nemacs-*= transport directory |
| =NEMACS_BRIDGE_BACKEND= | =session= | =session= / =nelisp= / =auto= |
| =NEMACS_BRIDGE_SOURCE= | =nelisp-emacs/src/nemacs-gui-file-bridge-runtime.el= | runtime image source |
| =NEMACS_RUNTIME_IMAGE= | =nelisp-emacs/build/nemacs-gui-file-bridge.nlri= | generated runtime image |
| =NEMACS_SYNC_NELISP= | =0= | =1= なら起動/build 前に snapshot 同期 |
| =NEMACS_USER_BIN= | =~/.local/bin= | =install-user-bin.sh= の symlink install 先 |
| =NEMACS_INSTALL_MANIFEST= | =~/.local/share/nemacs/install-manifest= | install / rollback の管理 manifest |

=NEMACS_TRANSPORT_DIR= は =nemacs-mx.sh=、=nelisp-emacs= runtime/session bridge、
および =nemacs-build.sh= で compile する native GUI binary の
=nemacs-*= transport directory を変える。native binary は実行時に環境変数を読まないため、
非 default transport は build-time rewrite と launcher の compile marker で一致させる。
=/tmp/nemacs.cfg= と =/tmp/nemacs-win.bin= は現時点では固定 path のまま。

bridge backend:
| backend | 動作 |
|---------+------|
| =session= | =nelisp-emacs= runtime image を常駐させ、=/tmp/nemacs-session-request= / =response= で同期する |
| =nelisp= | command ごとに runtime image を使う。runtime image 未準備なら失敗 |
| =auto= | session backend を優先し、失敗時は runtime image に落とす。host fallback は使わない |

=bin/emacs= は対話 GUI 起動を nemacs に向ける。=+LINE[:COL]= は launcher が byte offset に解決し、
native 起動時の初期 point に反映する。=--eval= / =--load= / =-l= / =--script= /
=--batch= / =-f= は =nelisp-emacs/bin/nemacs --driver=nelisp= で実行する。
=-nw= は =nelisp-emacs= の terminal UI を起動し、=--daemon= / =--fg-daemon= は
=nelisp-emacs/bin/nemacs-server= の emacsclient server を起動する。
install は =~/.local/bin/emacs= / =nemacs= symlink と manifest だけを管理し、
rollback は manifest と一致する symlink だけを削除する。問題発生時は
=/usr/bin/emacs FILE=、=PATH=/usr/bin:/bin:$PATH emacs FILE=、または
=dev/nelisp-gui/scripts/install-user-bin.sh rollback= で回避する。
起動 / install UX の詳細は =docs/design/07-launch-install-ux.org= と
=docs/design/14-install-rollback-plan.org= を参照する。

* repo / workspace 方針
repo は分けたまま、開発時は sibling checkout として並べる。

#+begin_src text
dev/
  nelisp/        Layer 1 runtime / reader / AOT compiler
  nelisp-emacs/  Layer 2 Emacs semantics / command substrate
  nelisp-gui/    Layer 3 native GUI display/input adapter
#+end_src

=nelisp-emacs= には vendor がある。ただし意味は二つに分かれる。
=nelisp-emacs/vendor/nelisp= は =nelisp-emacs= 本体ではなく、=make nelisp= /
=bin/build-nelisp= が作る NeLisp core runtime の optional な依存 checkout。
=nelisp-emacs/vendor/emacs-lisp= は upstream Emacs Lisp を取り込んで互換化する対象であり、
active な =nelisp-emacs= repo そのものではない。共同開発 workspace は sibling の
=dev/nelisp= / =dev/nelisp-emacs= / =dev/nelisp-gui= であり、vendor 配下ではない。
=nelisp-emacs= と =nelisp-gui= はどちらも first-class repo として commit history を分ける。
日常開発の導線は =NELISP_ROOT= / =NEMACS_EMACS_ROOT= を未指定にして sibling repo を自動検出する形を標準にする。

workspace の状態確認:
#+begin_src sh

#+end_src

cross-repo 検証は default =/tmp/nemacs-*= transport を共有する smoke が残るため、
現時点では逐次実行する。

* 責務境界
最終ルール: **nelisp-emacs は Emacs の意味論、nelisp-gui は表示/input adapter**。

| 判断対象 | owner | 理由 |
|----------+-------+------|
| buffer 内容、point、mark、region、kill-ring、undo、modified flag、visited file、buffer list、window layout/selection/window-start/window-hscroll/window-split-delta/tab-state/frame-state の変化 | =nelisp-emacs= | Emacs command の結果そのもの |
| keymap lookup、=command-execute=、=call-interactively=、=execute-extended-command=、minibuffer completion/history/require-match | =nelisp-emacs= | Emacs の command loop 意味論 |
| =find-file=、=save-buffer=、=write-file=、=self-insert-command=、movement/editing/kill/yank/window/help/search/undo command | =nelisp-emacs= | 同じ key/command で何が起きるかを決める層 |
| X11 socket/wire protocol、window id、pixmap、GC、font metrics、色、glyph/cursor/modeline/minibuffer の描画 | =nelisp-gui= | 表示 backend 固有の実装 |
| X11 event decode、raw key sequence 生成、=/tmp/nemacs-*= transport 初期化、runtime image/session 起動 | =nelisp-gui= | GUI と Layer 2 をつなぐ adapter |
| smoke test の焦点 | =nelisp-gui= は入力→transport→描画、=nelisp-emacs= は command 意味論 | 失敗時の修正先を分けるため |

禁止事項:
- =nelisp-gui= に Emacs command の簡易実装を追加しない。
- GUI 側で返却 snapshot の point/window-start/modeline/minibuffer state を補正しない。
- key binding を GUI 側の command branch として増やさない。raw key を渡し、lookup/dispatch は =nelisp-emacs= に置く。
- host Emacs fallback は migration 用であり、所有権を変えない。fallback で必要性が見えた command は =nelisp-emacs= に移す。

bridge 契約:
- GUI → =nelisp-emacs=: =/tmp/nemacs-keys= の raw key sequence と、現在の snapshot
  （buffer、point、mark、kill-ring mirror、buffer name、visited file、window layout、
  selected window、window-start、window-hscroll、window-split-delta、tab-state、frame-state、read-only など）を渡す。
- =nelisp-emacs= → GUI: 更新済み snapshot、point、mark、kill-ring mirror、buffer/window state、
  tab-state、frame-state、modeline、cursor、minibuffer active/prompt/state/candidates/history/require-match、status/error を返す。
- =/tmp/nemacs-cmd= / =/tmp/nemacs-arg= / =/tmp/nemacs-minibuffer-text= は direct fallback /
  migration transport。新しい daily-driver 操作は raw key/session path を優先する。

この repo は **GUI 表示・入力アダプタのみ**を担当する。Emacs の編集意味論と
command 実装は =nelisp-emacs= を使う。

短い結論:
- 「同じ command を実行した時に buffer / point / mark / file / kill-ring / undo がどう変わるか」は
  =nelisp-emacs= の責務。
- 「X11 入力を raw key sequence として transport し、返ってきた minibuffer / buffer 状態を
  どう描画するか」は =nelisp-gui= の責務。keymap lookup / command dispatch は
  =nelisp-emacs= の責務。
- =nelisp-gui= に Emacs command の簡易実装を増やさない。暫定 spike を入れた場合も、
  置き換え対象として固める時点で =nelisp-emacs= へ移し、bridge 経由に戻す。

この repo で実装してよいのは「見た目」「入力 decode」「bridge transport」だけ。
=execute-extended-command=, =find-file=, =save-buffer=, =self-insert-command=, =backward-delete-char=,
=newline=, =undo=, =other-window= などの command 結果はすべて
=nelisp-emacs= が決める。GUI は raw key sequence と現在の snapshot を渡し、
戻ってきた snapshot をそのまま描く。=/tmp/nemacs-cmd= の直接 command bridge は
未移行経路の fallback / migration path として残し、=auto= ではまず常駐 session に渡す。

| repo | 責務 | ここで扱う例 |
|------+------|--------------|
| =dev/nelisp= | Layer 1 runtime / reader / AOT compiler / standalone binary | =target/nelisp= の build、=/tmp/nelisp-snap= 同期 |
| =dev/nelisp-emacs= | Layer 2 Emacs substrate / interactive command / buffer-file state | =find-file=, =save-buffer=, =write-file=, =command-execute=, keymap/minibuffer |
| =dev/nelisp-gui= | Layer 3 GUI backend / X11 wire / frame paint / key/minibuffer transport | X11 描画、フォント、色、カーソル、minibuffer 表示、=/tmp/nemacs-*= snapshot transport |

判断基準は以下。

| 置き場所 | 入れるもの | 入れないもの |
|----------+------------+--------------|
| =nelisp-emacs= | buffer 内容、point、mark、visited file、modified 状態、kill-ring、undo、Emacs window state、command dispatch、Emacs Lisp API 互換 | X11 socket、描画ループ、フォント/色の実描画、window-system 固有 transport |
| =nelisp-gui= | frame paint、glyph/cursor 表示、X11 event decode、minibuffer の表示と入力収集、snapshot/point/key/minibuffer/window display transport | =find-file= / =save-buffer= / =write-file= / movement / editing / window command の意味論、keymap lookup |

=nelisp-gui= 側で =find-file= 等の動作を再実装しない。GUI は現在の buffer snapshot、
point、raw key sequence を bridge に渡し、bridge は =nelisp-emacs/src/files.el=
などの command 実装を呼んで結果 snapshot と point を GUI に戻す。未接続の command は暫定で
host Emacs fallback に逃がす。現時点の nelisp backend は =nelisp-emacs= 側の
GUI bridge runtime adapter にある =command-execute= / =call-interactively= を通して
command を実行する。

開発時のルール:
- 新しい =M-x= / key binding / file command / editing command は、まず
  =nelisp-emacs= に command と状態更新を置き、=nelisp-gui= から raw key bridge で呼ぶ。
- =nelisp-gui= の直接 buffer 変更は spike / 低遅延表示の暫定処理に限る。置き換え対象の
  Emacs command として固める時点で =nelisp-emacs= 側へ移す。
- GUI に key binding を追加する時は、=nelisp-emacs= 側に command 実装と意味論テストがあることを
  前提にし、この repo では key event decode、transport、再描画 smoke だけを見る。
- key binding の移植順は、=nelisp-emacs= で
  =make nemacs-gui-keymap-coverage= を実行して host Emacs の =global-map= と
  GUI bridge runtime keymap の差分を見て決める。=missing= は次の command 実装候補、
  =command-missing= は keymap だけ先に入って command が未実装の状態として扱う。
  =different= は host Emacs と runtime の keymap 名がずれた状態として扱う。互換 alias がある場合も、
  raw key lookup は host Emacs と同名に寄せ、command 意味論テストで alias の同等性を確認する。
- =nelisp-gui= のテストは「入力 → transport → 再描画」を見る。command の意味論テストは
  =nelisp-emacs/test/nemacs-gui-file-bridge-runtime-test.el= に置く。
- host Emacs fallback は未移植 command の保険であり、所有権は fallback 先ではなく
  =nelisp-emacs= 側にある。

hard boundary:
- =nelisp-gui= は「X11 入力を raw key sequence に decode して運び、返却された minibuffer state を描く」までを担当する。
  keymap lookup、command dispatch、command の実行結果を決めるロジックは持たない。
- =find-file=, =save-buffer=, =write-file=, =revert-buffer=, movement/editing/region/kill/undo
  はすべて =nelisp-emacs= の command として扱う。GUI 側は raw key sequence、
  snapshot、point/mark/kill-ring mirror を transport するだけにする。
  =/tmp/nemacs-cmd= / =/tmp/nemacs-arg= は direct command fallback のためだけに使う。
- GUI 都合の状態は =nelisp-gui= が持つ。例: X11 window id、pixmap、GC、font metrics、
  dirty rectangle、cursor blink、minibuffer 表示文字列、key prefix 入力中フラグ。
- Emacs buffer 都合の状態は =nelisp-emacs= が持つ。例: visited file、modified flag、
  point、mark、region、kill-ring、undo、current buffer contents、current buffer name、
  buffer ごとの point / mark / window-start、window layout / selected window /
  window-start の command 意味論、command dispatch。
- =nelisp-gui= に Emacs command の簡易版を増やさない。即時描画のための暫定処理を入れた場合も、
  同じ振る舞いを =nelisp-emacs= に移して bridge 経由に戻す。

bridge 契約:
- GUI → =nelisp-emacs=: raw key sequence（=/tmp/nemacs-keys=）、buffer snapshot、point、mark、kill-ring mirror、
  buffer name、window layout、selected window、window-start、window-hscroll、window-split-delta、tab-state、frame-state を渡す。
  =/tmp/nemacs-cmd= / =/tmp/nemacs-arg= は direct command bridge の fallback / migration path。
- =nelisp-emacs= → GUI: 更新済み buffer snapshot、point、mark、kill-ring mirror、
  buffer name、window layout、selected window、window-start、window-hscroll、window-split-delta、tab-state、frame-state、modeline、cursor、
  minibuffer active/prompt/state/candidates/history、
  status/error を返す。
- GUI は返された snapshot を描画する。返却前に command の意味論を補正しない。
- 未対応 command は「GUI未実装」ではなく「=nelisp-emacs= command/bridge未実装」として扱う。

実装判断チェック:
- X11/描画/フォント/イベント decode の変更なら、この repo。
- =find-file= や編集結果がどう変わるかの変更なら、=nelisp-emacs=。
- key binding 追加なら、command 本体、keymap lookup、意味論テストは =nelisp-emacs=、
  X11 key decode、transport、再描画 smoke はこの repo。
- minibuffer の見た目や入力収集はこの repo。minibuffer で呼ばれる command の意味論、
  completion 候補、buffer/file 状態更新は =nelisp-emacs=。
- window 分割線・選択枠・scroll 後の描画開始位置をどう描くかはこの repo。

この bridge 専用 adapter を =nelisp-emacs/src/emacs-command-loop.el= /
=src/emacs-fileio.el= 本来の runtime image に統合すること。
=window selected= は =nelisp-emacs= が決めた選択 window index を GUI が選択枠として描くための表示状態で、
選択遷移の意味論は =other-window= / =delete-window= command として =nelisp-emacs= 側に置く。
=nelisp-emacs= 側には =test/nemacs-gui-file-bridge-runtime-test.el= があり、
=make test-nemacs-gui-bridge= で standalone reader 上の bridge dispatch を検証する。

* 実装済み機能（全て native 機械語、自動検証済）
設定駆動 geometry / 最大化 / フルスクリーン、フォント指定 + リアルタイム文字サイズ、
複数行対話編集（挿入/BS/改行、カーソル ←→↑↓ 列保持）、構文 + org-mode ハイライト、

=nelisp-emacs= 経由のファイル I/O。

* native IR の制約（aot compiler）
3引数 =+= 不可（ネスト）/ =cond= は値として test を返す（ネスト =if=）/ =alloc-bytes= は
deref 不可（=mmap= = syscall 9 を使う）/ =if= 条件の =ptr-read= は compiler が fold → ローカル束縛。
