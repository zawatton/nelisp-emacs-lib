* Classification Rules

- *Immediately implementable* means no new GUI rendering primitive, process
  substrate, or large package surface is the main blocker.  These still need
  runtime semantic tests and a bridge smoke.
- *GUI drawing dependent* means the command changes visible frame/window/tab,
  text-scale, rectangle/selection, or display target state that the GUI must
  render correctly.
- *Process/syscall dependent* means useful behavior needs process creation,
  pipes/stdin/stdout/stderr, async process events, external tools, network, or
  VC subprocesses.
- *Major-mode/package dependent* means the missing binding is mainly blocked
  by a larger Emacs Lisp package, mode-specific indexer, input/coding system,
  mail, project, abbrev, or imenu subsystem.

When a command touches multiple areas, it is listed under the first blocker
that would make a user-visible smoke meaningful.
