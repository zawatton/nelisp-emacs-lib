#+title: GUI Transport State Contract
#+date: 2026-06-09

* Purpose

This document fixes the ownership and lifecycle of the =nemacs-*= file
transport shared by =nelisp-gui= and =nelisp-emacs=.  It is a contract for
parallel development: command semantics stay in =nelisp-emacs=, while
=nelisp-gui= initializes transport, sends raw input, starts the bridge, and
renders the returned snapshot.

The default transport directory is =/tmp=.  =NEMACS_TRANSPORT_DIR= may point
=nemacs-mx.sh=, the runtime/session bridge, and build-time native
path rewriting at another directory.  =/tmp/nemacs.cfg= and the X11 socket path
are not transport state and remain fixed outside this contract.

* Ownership Rules

| area | owner | rule |
|------+-------+------|
| command result semantics | =nelisp-emacs= | Owns buffer text, point, mark, file state, command dispatch, keymap lookup, minibuffer behavior, kill-ring, registers, bookmarks, window/tab semantic state, and status/error results. |
| input adapter | =nelisp-gui= | Owns X11 event decoding, raw key sequence generation, launcher initialization, session startup, and direct fallback invocation. |
| rendering | =nelisp-gui= | Reads returned snapshot and paints it.  It must not repair point, window-start, hscroll, modeline, minibuffer, tab, or buffer state. |
| migration fallback | shared but temporary | =nemacs-cmd= / =nemacs-arg= / direct minibuffer files remain for unmigrated command paths.  Daily-driver behavior should prefer =nemacs-keys= and session runtime. |
| tests | both repos | Default =/tmp/nemacs-*= tests must hold =/tmp/nemacs-transport.lock= or run alone.  Isolated transport tests should use =NEMACS_TRANSPORT_DIR=. |

* Lifecycle Classes

| class | meaning | examples |
|-------+---------+----------|
| launch-initialized | Written by =bin/nemacs= before the native GUI starts.  It prevents stale state from an older session from being drawn. | =nemacs-cmd=, =nemacs-keys=, =nemacs-minibuffer-active=, =nemacs-window-layout=, stores |
| per-command input | Written by GUI/native or smoke tests before =nemacs-mx.sh= / runtime runs. | =nemacs-keys=, =nemacs-cmd=, =nemacs-arg=, =nemacs-minibuffer-text= |
| returned snapshot | Written by =nelisp-emacs= runtime after command execution, then rendered by =nelisp-gui=. | =nemacs-buf=, =nemacs-point=, =nemacs-modeline=, =nemacs-cursor= |
| session control | Written by =nemacs-mx.sh= and the persistent runtime to synchronize requests. | =nemacs-session-request=, =nemacs-session-response=, =nemacs-session-ready= |
| durable within launch | Preserved across commands in one GUI launch/session, but reset by launcher startup or test setup. | kill-ring, buffer stores, registers, bookmarks, undo, kmacro/session state |
| derived/test-only | Temporary probe, smoke output, or verifier artifact.  Do not make runtime behavior depend on it. | =nemacs-visual-*.xwd=, =nemacs-*-test.txt=, =nemacs-bridge-nelisp-probe= |

* Protocol State Table

Paths below mean =$NEMACS_TRANSPORT_DIR/NAME=, defaulting to =/tmp/NAME=.
Fixed =/tmp= files are called out explicitly.  "Both" means GUI/test code may
seed the file before a command, but runtime/session bridge owns the semantic
result written back after dispatch.

* Stale-State Rules

- Launcher startup must reset input, minibuffer display, modeline/cursor, window
  state, buffer stores, registers, bookmarks, kill-ring, rectangle, text-scale,
  prefix, kmacro, and read-only state before native GUI paint.
- Raw keys have priority over direct command fallback.  If =nemacs-keys= is
  non-empty, =nemacs-cmd= is stale and must be cleared before dispatch.
- =nelisp-gui= must not clamp or correct returned =nemacs-point=,
  =nemacs-window-start=, =nemacs-window-hscroll=, =nemacs-modeline=, or
  minibuffer state.  Bugs in those values belong to =nelisp-emacs=.
- Session tests and GUI verifier runs must not share default =/tmp= transport
  concurrently.  Use =NEMACS_TRANSPORT_DIR= for isolated smokes, or rely on the
  shared =nemacs-transport.lock= for fixed-path tests.
- Temporary files whose names merely start with =nemacs-= but include test names,
  screenshots, probes, or stdout/stderr captures are diagnostics, not protocol.

* Extension Checklist

When adding a new =nemacs-*= state file:

1. Record the owner in this document before depending on it in two repos.
2. Initialize it in =bin/nemacs= if stale content can affect first paint or the
   first command.
3. Thread =NEMACS_TRANSPORT_DIR= through =nemacs-mx.sh= and runtime code; do not
   hard-code =/tmp= unless the file is intentionally outside transport.
4. Add runtime tests in =nelisp-emacs= for command semantics.
5. Add GUI verifier coverage only for input transport or rendering behavior.
6. Keep default =/tmp= tests serialized by =nemacs-transport.lock=.
