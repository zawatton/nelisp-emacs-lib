* Classification Rules

Each missing command gets one primary bucket:

| Bucket | Meaning |
|--------+---------|
| Immediate | Can be implemented with existing buffer/window/minibuffer state and focused smoke tests. |
| GUI/display dependent | Needs new rendered state, frame/tab/window UI, narrowing display behavior, text scaling, or highlight/font-lock display. |
| Process/syscall dependent | Needs fork/exec/pipe/wait, subprocess lifetime, shell quoting, or external tools. |
| Major-mode/package dependent | Needs a larger Emacs subsystem such as kmacro, abbrev, project, vc, input methods, coding systems, mail, ispell, imenu, or EWW. |

Some commands overlap buckets.  For example =vc-diff= is both VC package work
and process work; it is listed as process/syscall first because the command is
not useful until external process execution is reliable.
